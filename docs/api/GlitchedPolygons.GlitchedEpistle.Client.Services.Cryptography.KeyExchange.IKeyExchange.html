Interface IKeyExchange

This key utility was extracted first of all, obviously, to avoid code duplication and provide a single point of exchange where API consumers can prepare user RSA keys for submission to the backend and, conversely, deserialize them back into the client's domain model easily.

Secondly, it's also some sort of code entry point for whoever wants to verify Epistle's safety: you can Shift-F12/navigate into the various message exchange/encryption relevant implementations from here and convince yourself whether this product is for you or not, by reading the source code.

EncryptAndCompressPrivateKey(String, String)

Encrypts privateKeyPem into a portable string that is safe to exchange with the backend, AS LONG AS THE USER'S PASSWORD IS NOT COMPROMISED.

THE SERVER NEVER HAS YOUR PRIVATE KEY IN PLAIN TEXT!

The private key is encrypted using the user's password, NOT ITS HASH! Otherwise the server could decrypt the key, because it does have the user's PasswordSHA512.

Because of this, it is highly advisable to change user passwords often and use an offline, open-source password manager (such as KeePass).
